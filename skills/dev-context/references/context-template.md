## 各セクションの記述ガイド

### Tech Stack
- パッケージ定義ファイルから直接読み取った情報は 🔵
- コードから推定した情報は 🟡
- バージョンが特定できない場合は「latest」ではなく「unknown」と記載

### Test Framework
- テスト実行コマンドは **必ず記載する**（後続スキルが依存する最重要情報）
- 複数のテストフレームワークがある場合はすべて列挙
- E2Eテストと単体テストは分けて記載

### Project Structure
- 深さは2-3階層まで
- 空ディレクトリや設定ファイルのみのディレクトリは省略
- コメントで各ディレクトリの役割を簡潔に記述

### Coding Conventions
- 代表的なファイルから帰納的に推定
- 確証がない場合は 🟡 を付与
- プロジェクトで明示的な設定（.eslintrc等）がある場合はそれを優先

### Key Entry Points（リポマップ）
- 主要なモジュール・サービス・コンポーネントを10-20個
- ファイルパスは相対パス
- 役割は1文以内

### Build & Run
- scripts/ の内容がある場合はそれを優先
- 検出できたコマンドのみ記載（推測しない）

## 行数制限

context.md 全体で **500行以内** に収める。以下の優先順位で情報を圧縮する:

1. Tech Stack / Test Framework / Build & Run: 必須（圧縮不可）
2. Project Structure / Key Entry Points: 重要（ディレクトリ深度やモジュール数で調整）
3. Coding Conventions: 主要なものだけ残す
4. Additional Notes: 最小限に
